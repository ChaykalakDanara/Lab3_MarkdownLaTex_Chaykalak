# Лабораторная работа №3. Работа с Markdown-разметкой и базовое использование LaTeX в документации проекта

#### Markdown (Маркдаун) — это легкий язык разметки для форматирования текста с помощью простых символов (вроде * для курсива и ** для жирного шрифта), созданный для удобства чтения и написания, который легко преобразувать в HTML или другие форматы, широко используется на GitHub, Reddit, в документации и редакторах. Его главная цель — сделать написание структурированного текста интуитивно понятным, не требуя сложных тегов, и сохранить читаемость даже в исходном виде. 
#### Markdown используют в системах управления контентом (CMS), редакторах кода, вики-платформах, а также на GitHub, Stack Overflow и других ресурсах, где важна удобная разметка текста. Его применяют для документации, статей, заметок, блогов и даже электронных книг.
#### Разработчикам Markdown нужен повсеместно: для README-файлов в репозиториях (GitHub, GitLab), технической документации к проектам, заметок (Obsidian, Notion), постов и комментариев на Stack Overflow, а также в CMS, генераторах сайтов (Jekyll) и даже мессенджерах (Telegram, Discord) для форматирования текста, поскольку он прост, читаем и легко конвертируется в HTML.
